### Markdown

大部分内容来自[Markdown中文网](http://markdown.p2hp.com/)  

Markdown是一种简单易用的标记语言,由John Gruber设计，您可以使用它来格式化几乎任何文档。

#### 基本语法：

几乎所有Markdown应用程序（如上面提到的[Obsidian](https://obsidian.md/)、[Microsoft Visual Studio Code](https://code.visualstudio.com/)、[Typora](https://typoraio.cn/)、[MarkText](https://www.github.com/marktext/marktext)）都支持John Gruber原始设计文档中概述的基本语法，而且渲染样式基本相同  
Tip：

> 您可以将HTML标签添加到任何Markdown文件中。如果您更喜欢某些HTML标记而不是Markdown语法，这将很有帮助。例如，有些人发现将HTML标签用于图像更容易。

#### 扩展语法：

John Gruber的原始设计文档中概述的基本语法每天增加了许多所需的元素，有时候我们会发现这还不够（比如当你需要键入一些奇怪的化学符号的时候）。这就是扩展语法的用武之地。

一些个人和组织开始通过添加表，代码块，语法突出显示，URL自动链接和脚注等其他元素来扩展基本语法。可以通过使用基于基本Markdown语法的轻量级标记语言，或者通过向兼容的Markdown处理器添加扩展来启用这些元素。

可用性  
并非所有Markdown应用程序都支持扩展语法元素。您需要检查您的应用程序所使用的轻量级标记语言是否支持您要使用的扩展语法元素。如果没有，仍然可以在Markdown处理器中启用扩展。  

详细内容可以访问[Markdown中文网](http://markdown.p2hp.com/) ，如果您希望自己可以帮助我们，您可以在这里找到相当不错的教程，然后开始您的学习之旅(注：Markdown中文网里Github上的一些文档可以不看)。我们推荐您完成基本语法和扩展语法后继续往下。

### LaTeX

下列内容部分来自[百度百科-LaTeX](https://baike.baidu.com/item/LaTeX/1212106)  

LaTeX（LATEX，音译“拉泰赫”）是一种基于ΤΕΧ的排版系统，由美国计算机学家Leslie Lamport在20世纪80年代初期开发，利用这种格式，即使使用者没有排版和程序设计的知识也可以充分发挥由TeX所提供的强大功能，能在几天、甚至几小时内生成很多具有书籍质量的印刷品。对于生成复杂表格和数学公式，这一点表现得尤为突出。因此它非常适用于生成高印刷质量的科技和数学类文档。这个系统同样适用于生成从简单的信件到完整书籍的所有其他种类的文档。

在我们这个比较学术性的文档中，很多超越纯文本的排版离不开LaTeX的帮忙，详情可以访问[LaTeX的官网](https://www.latex-project.org/)

LaTeX的相关学习可以参考[LaTeX语法速查表](https://www.jianshu.com/p/7efccfc5f5a2)、[LaTeX数学公式大全](https://www.luogu.com.cn/blog/IowaBattleship/latex-gong-shi-tai-quan)和[速查](Quickcheck.md)

注意：

> 有些LaTeX+Markdown原生语法没法正常在Obsidian中显示，因为VScode，我们以[Typora](https://typoraio.cn/)和[Microsoft Visual Studio Code](https://code.visualstudio.com/)安装[Markdown All in One](https://marketplace.visualstudio.com/items?itemName=yzhang.markdown-all-in-one)、[LaTeX Workshop](https://marketplace.visualstudio.com/items?itemName=James-Yu.latex-workshop)和[LaTeX Language Support](https://marketplace.visualstudio.com/items?itemName=torn4dom4n.latex-support)扩展为准。
> 
> 导出PDF以Typora和Marktext为主

## 我希望参与编辑工作

欢迎！我们只是学生，笔记部分一定会有所纰漏，欢迎各位奆佬的鼎力相助！我们也缺很多劳动力，包括优化排版及表述、以及SVG图片、有机化学排版的相关帮助！  

鉴于本项目所使用的所有软件在Windows，Linux和MacOS下都可以运行，所以理论上来说你不需要任何对自己系统的进一步配置。

您可以这么做：  

1. 把这个仓库克隆到本地  
   
   ```git
     git clone https://gitee.com/Situ-Chengxiang/note-sync.git
   ```

2. 配置Git和Gitee
   
   - 会的直接往下跳，不会的参考一下[菜鸟教程](https://www.runoob.com/git/git-gitee.html) 
     
     1. 注册gitee账户
     
     2. 生成ssh密匙
     
     3. 下载git并配置（Linux应该大部分自带git了，Windows和MacOS用户请自行下载）

3. 下载您的公式编辑器
   
   #### Typora
   
   许多人鼎力推荐的markdown编辑器，优雅美观，符合使用习惯，不过现在收费
   
   #### MarkText
   
   自从Typora收费之后，很多人一直在找免费的markdown编辑器，这里个人推荐一下Marktext，的确相当棒，而且带图形化提示，非常适合新手。
   
   值得一提的有时候渲染其他软件写的可能会加载不出来，稍微调整一下就行。
   
   #### Microsoft Visual Studio Code
   
   Microsoft Visual Studio Code是一款强大的编辑器，和其他几款专门编辑markdown的不同，这本来是一款代码编辑器，
   
   1. 下载安装[Microsoft Visual Studio Code](https://code.visualstudio.com/)(有时候我们喜欢称之为VScode)
   
   2. 在扩展商店里安装[Git]()、[Markdown All in One](https://marketplace.visualstudio.com/items?itemName=yzhang.markdown-all-in-one)、[LaTeX Workshop](https://marketplace.visualstudio.com/items?itemName=James-Yu.latex-workshop)和[LaTeX Language Support](https://marketplace.visualstudio.com/items?itemName=torn4dom4n.latex-support)四个扩展

4. 安装[Obsidian](https://obsidian.md/)

5. 安装LaTeX库作为辅助，我们推荐[MikLaTeX](https://miktex.org/)，相对来讲这玩意体量小不少（完整版会有2~3GB，MikLaTeX可以根据需要下载所需要的渲染引擎）

6. 打开Obsidian，选择库的时候选择克隆下来的文件夹

7. 用Obsidian开始预览，找到您感兴趣的地方，进入查看，如果渲染异常，将这个文件用其他编辑器打开试试（新手首选MarkText），如果有所错误或缺漏，您可以帮我们指正补全，修改结束后，保存编辑，然后回到文件夹根目录（带有.gitgnore文件的文件夹），然后用打开cmd（或者Windows Termianl之类的终端），cd进文件夹，然后依次输入  `git add .`  
   
   `git push `

8. 您的更改将于审核后合并（不超过24h）

9. 每次启动前记得在文件夹内打开终端，然后输入
   
   `git pull`
   
   以查看最新的更改，值得一提的VSCode作为代码编辑器这方面做得非常好，**我们推荐有能力的各位慢慢从Marktext和Typora过渡到习惯与VSCode和Git构建起的开发环境**，因为Marktext和Typora的很多功能可以借助VSCode和其强大的扩展以实现。

Tip：

- 我们其实不推荐一个人同时去负责很多门学科的编辑工作，真的会比较累，每个人只需要注重自己所负责的一块内容就行了，大家合作在后面修修补补，加上我们的审核团队，应该足以满足需求了。

- 学术性问题的讨论请移步[*Firprin*](https://www.firprin.com/)，记得遵循社区守则

- 更多编辑须知见下
  
  #### 编辑工作须知
  
  1. 学术性交流需要是开放、尊重、理性、和谐
  
  2. 文件命名原则如下：那门课哪本书-第几章-第几节
     
     如：物理必修第三册-第十三章-电磁波与电磁感应初步-第1节-电磁波  
  
  3. 文件中的排标题方式：
     
     - 一级标题（H1）：本节名称（比如电磁波）
     
     - 二级标题（H2）：本小节名称
     
     - 三级标题（H3）：你觉得需要标题的地方
     
     - 以此类推可以写到6个
  
  4. 公式拿`$\boxed$`括起来（去看[LaTeX & Markdown完全指南](https://www.cnblogs.com/Rosmarinus/articles/15553532.html)），数学物理需要长篇大论的记得等号对齐（或者其他好看规范的排法，我觉得教科书是个非常好的例子）
  
  5. 需要加入视频资源的放个链接摆那就行，你觉得flame好使也可以试试
  
  6. **图片传到图床上，可以不写描述，但是最好吧link带上。不推荐直接作为文件塞到附件文件里（CDN很多，找家靠谱点的！）**
  
  7. 高中阶段的统一去改人教版的，更新的时候如果发现不一样的内容，或需要补充，请在原文下像这样输入
     
     > 这是一段和你教材里不一样的文段
     > 
     > 空一行
     > [你的教材版本-第几本书--第几章-第几节-**小标题或页码**]：
     > 
     > 比如：[浙科版-生物必修第二册-第一章-P12]
     > 
     > 下面是你需要修改或者补充的内容
