### Markdown

大部分内容来自[Markdown 中文网](http://markdown.p2hp.com/)

Markdown 是一种简单易用的标记语言,由 John Gruber 设计，您可以使用它来格式化几乎任何文档。

#### 基本语法：

几乎所有 Markdown 应用程序（如上面提到的[Obsidian](https://obsidian.md/)、[Microsoft Visual Studio Code](https://code.visualstudio.com/)、[Typora](https://typoraio.cn/)、[MarkText](https://www.github.com/marktext/marktext)）都支持 John Gruber 原始设计文档中概述的基本语法，而且渲染样式基本相同  
Tip：

> 您可以将 HTML 标签添加到任何 Markdown 文件中。如果您更喜欢某些 HTML 标记而不是 Markdown 语法，这将很有帮助。例如，有些人发现将 HTML 标签用于图像更容易。

#### 扩展语法：

John Gruber 的原始设计文档中概述的基本语法每天增加了许多所需的元素，有时候我们会发现这还不够（比如当你需要键入一些奇怪的化学符号的时候）。这就是扩展语法的用武之地。

一些个人和组织开始通过添加表，代码块，语法突出显示，URL 自动链接和脚注等其他元素来扩展基本语法。可以通过使用基于基本 Markdown 语法的轻量级标记语言，或者通过向兼容的 Markdown 处理器添加扩展来启用这些元素。

可用性  
并非所有 Markdown 应用程序都支持扩展语法元素。您需要检查您的应用程序所使用的轻量级标记语言是否支持您要使用的扩展语法元素。如果没有，仍然可以在 Markdown 处理器中启用扩展。

详细内容可以访问[Markdown 中文网](http://markdown.p2hp.com/) ，如果您希望自己可以帮助我们，您可以在这里找到相当不错的教程，然后开始您的学习之旅(注：Markdown 中文网里 Github 上的一些文档可以不看)。我们推荐您完成基本语法和扩展语法后继续往下。

### LaTeX

下列内容部分来自[百度百科-LaTeX](https://baike.baidu.com/item/LaTeX/1212106)

LaTeX（LATEX，音译“拉泰赫”）是一种基于 ΤΕΧ 的排版系统，由美国计算机学家 Leslie Lamport 在 20 世纪 80 年代初期开发，利用这种格式，即使使用者没有排版和程序设计的知识也可以充分发挥由 TeX 所提供的强大功能，能在几天、甚至几小时内生成很多具有书籍质量的印刷品。对于生成复杂表格和数学公式，这一点表现得尤为突出。因此它非常适用于生成高印刷质量的科技和数学类文档。这个系统同样适用于生成从简单的信件到完整书籍的所有其他种类的文档。

在我们这个比较学术性的文档中，很多超越纯文本的排版离不开 LaTeX 的帮忙，详情可以访问[LaTeX 的官网](https://www.latex-project.org/)

LaTeX 的相关学习可以参考[LaTeX 语法速查表](https://www.jianshu.com/p/7efccfc5f5a2)、[LaTeX 数学公式大全](https://www.luogu.com.cn/blog/IowaBattleship/latex-gong-shi-tai-quan)和[速查](Quickcheck.md)

>

鉴于本项目所使用的所有软件在 Windows，Linux 和 MacOS 下都可以运行，所以理论上来说你不需要任何对自己系统的进一步配置。

   #### Typora

   许多人鼎力推荐的 markdown 编辑器，优雅美观，符合使用习惯，不过现在收费

   值得一提的有时候渲染其他软件写的可能会加载不出来，稍微调整一下就行。

   #### Microsoft Visual Studio Code

   Microsoft Visual Studio Code 是一款强大的编辑器，和其他几款专门编辑 markdown 的不同，这本来是一款代码编辑器，

   1. 下载安装[Microsoft Visual Studio Code](https://code.visualstudio.com/)(有时候我们喜欢称之为 VScode)

- 学术性问题的讨论请移步[_Firprin_](https://www.firprin.com/)，记得遵循社区守则

- 更多编辑须知见下

  #### 编辑工作须知

  1. 学术性交流需要是开放、尊重、理性、和谐

  2. 文件命名原则如下：那门课哪本书-第几章-第几节

     如：物理必修第三册-第十三章-电磁波与电磁感应初步-第 1 节-电磁波

  3. 文件中的排标题方式：

     - 一级标题（H1）：本节名称（比如电磁波）

     - 二级标题（H2）：本小节名称

     - 三级标题（H3）：你觉得需要标题的地方

     - 以此类推可以写到 6 个

  4. 公式拿`$\boxed$`括起来（去看[LaTeX & Markdown 完全指南](https://www.cnblogs.com/Rosmarinus/articles/15553532.html)），数学物理需要长篇大论的记得等号对齐（或者其他好看规范的排法，我觉得教科书是个非常好的例子）

  5. 需要加入视频资源的放个链接摆那就行，你觉得 flame 好使也可以试试

  6. **图片传到图床上，可以不写描述，但是最好吧 link 带上。不推荐直接作为文件塞到附件文件里（CDN 很多，找家靠谱点的！）**

  7. 高中阶段的统一去改人教版的，更新的时候如果发现不一样的内容，或需要补充，请在原文下像这样输入

     > 这是一段和你教材里不一样的文段
     >
     > 空一行
     > [你的教材版本-第几本书--第几章-第几节-**小标题或页码**]：
     >
     > 比如：[浙科版-生物必修第二册-第一章-P12]
     >
     > 下面是你需要修改或者补充的内容
